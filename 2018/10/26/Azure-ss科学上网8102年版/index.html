    • Azure+ss科学上网8102年版

    此文是基于Azure搭建shadowsocks用作科学上网的补充，原文写于2015年，很多配置已经不适用了，所以特别在这里记录我近期搭建ss梯子过程中遇到的坑以及应对策略，建议结合食用效果更佳~

    0x00 创建虚拟机

    首先要在Azure上创建虚拟机，个人认为操作系统不会有什么影响，所以就用了默认的ubuntu server 18.04TLS。

    配置管理员账户

    为了安全起见，我用SSH公钥的方式登录admin账户。在cmd下输入

    默认会在C:\Users\username\.ssh下生成两个文件，id_rsa是私钥，id_public是公钥。

    在下图的位置输入用户名，再把id_public的内容复制粘贴即可。

    设置允许SSH

    需要注意的是，创建虚拟机的时候需要设置允许公共入站端口为SSH，不然接下来你会发现SSH怎么也连不上。(手动狗头)

    0x01 打开防火墙

    在侧边栏选择设置-网络，可以看到如下图，入站规则中有一条是我们初始化添加的用于SSH端口。但是还不够，需要再开一个用于ss通信的端口。

    选择“添加入站规则”，端口号可以为1-65525中的任何一个，但是不能和已有端口冲突。我这里设置的端口号是443，最好添加TCP和UDP两种协议。

    0x02 SSH连接虚拟机

    接下来的任务是用本地计算机登录虚拟机。

    我这里使用xshell完成这个操作，需要将第一步生成的rsa私钥导入。新建会话-输入虚拟机公网ip，右侧类别中选择连接-用户身份验证，在打开的对话窗口中选择导入，来到第一步生成的两个rsa密钥的文件夹，导入id_rsa文件。最后输入密码完成配置！

    到这里我们就可以成功连上虚拟机啦~
